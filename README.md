# Лабораторная работа №23. Интерактивные веб-приложения на JavaScript

### Описание игры

Логическая игра "Крестики-нолики" для двух игроков на поле 3×3. Цель игры — первым выстроить непрерывную линию из трёх своих символов по горизонтали, вертикали или диагонали.

#### Правила игры

- Игровое поле представляет собой сетку 3×3.
- Первый игрок играет крестиками ("X"), второй — ноликами ("O").
- Игроки по очереди ставят свои символы в пустые ячейки.
- Побеждает тот, кто первым выстроит три своих символа в ряд:
  - По горизонтали
  - По вертикали
  - По диагонали
- Если все 9 ячеек заполнены, но победитель не выявлен, объявляется ничья.

#### Логика победы/поражения

- **Победа**: Игра автоматически определяет победителя после каждого хода и выводит соответствующее сообщение.
- **Ничья**: При заполнении всех ячеек без выигрышной комбинации игра останавливается с сообщением "Ничья!".
- **Поражение**: Поражение не фиксируется отдельно — проигрывает тот, кто не смог выстроить ряд.

#### Особенности реализации

- Полностью интерактивный интерфейс без необходимости обновления страницы
- Визуальная обратная связь при наведении на клетки
- Отображение текущего статуса игры (чей ход, кто победил или ничья)
- Возможность начать новую игру без перезагрузки страницы
- Адаптивный дизайн с центрированным игровым полем

### Игровая механика

#### Как определяется победитель?

Победитель определяется функцией `checkWinner()`, которая:

1. Проходит по всем 8 возможным выигрышным комбинациям
2. Проверяет, заполнены ли все три клетки в комбинации одинаковым символом
3. Если условие выполнено, игра завершается, и отображается сообщение о победе

#### Как обрабатываются клики?

При клике на клетку:

1. Проверяется, активна ли игра (`gameActive === true`)
2. Проверяется, пуста ли выбранная клетка
3. Если условия выполнены, символ текущего игрока отображается в клетке
4. Обновляется массив `board`
5. Вызывается функция `checkWinner()` для проверки окончания игры
6. Если игра продолжается, происходит смена игрока

#### Как работает перезапуск?

Функция `restartGame()`:

1. Сбрасывает флаг `gameActive` в `true`
2. Устанавливает текущего игрока на "X"
3. Очищает массив `board` (заполняет пустыми строками)
4. Обновляет статус игры
5. Очищает все символы из клеток на странице

### Структура данных

**Массив board:**

Массив board состоит из 9 элементов, каждый из которых соответствует одной клетке игрового поля. Индексы от 0 до 8 представляют клетки в порядке слева направо и сверху вниз. Изначально все элементы содержат пустую строку (" "), что означает пустую клетку. Когда игрок делает ход, соответствующий индекс массива заменяется на "X" или "O". Этот массив служит источником правды о текущем состоянии игрового поля.

**Условия победы:**

Условия победы хранятся в массиве winningConditions, который содержит 8 подмассивов. Каждый подмассив содержит три индекса, представляющие выигрышную комбинацию: 3 горизонтали, 3 вертикали и 2 диагонали. Например, комбинация [0,1,2] означает верхнюю горизонтальную линию, а [0,4,8] — главную диагональ. Эти комбинации используются для проверки, заполнены ли три клетки одним и тем же символом.

**Состояние игры:**

Состояние игры хранится с помощью двух переменных: currentPlayer хранит символ текущего игрока ("X" или "O"), начиная с "X". gameActive — это булева переменная (true/false), которая указывает, продолжается ли игра. Если gameActive = false, игра завершена (победа или ничья), и новые ходы не обрабатываются. Также статус игры отображается в элементе statusText для пользователя.

### Алгоритмы

**Алгоритм проверки победы:**

- Функция checkWinner() перебирает все 8 выигрышных комбинаций из массива winningConditions.
- Для каждой комбинации получает значения из массива board по указанным индексам.
- Проверяет, не пустые ли эти клетки и все ли они равны между собой.
- Если найдена такая комбинация, переменная roundWon устанавливается в true.
- Если roundWon = true, игра завершается: статус обновляется сообщением о победителе, gameActive устанавливается в false, функция завершается.
- Если победы нет, игра продолжается или проверяется ничья.

**Алгоритм определения ничьи:**

- После проверки победы, если победитель не найден, функция проверяет, остались ли пустые клетки.
- Перебирает все элементы массива board в поисках пустой строки (" ").
- Если ни одного пустого элемента не найдено (все клетки заполнены), то объявляется ничья.
- Статус обновляется сообщением "Ничья!", gameActive устанавливается в false.

**Алгоритм смены игрока:**

- Если игра активна (gameActive = true) и не было победы или ничьи, вызывается функция смены игрока.
- Проверяется значение currentPlayer: если currentPlayer = "X", то он меняется на "O".
- Если currentPlayer = "O", то он меняется на "X".
- После смены игрока обновляется текст статуса, отображающий, чей сейчас ход.
- Этот алгоритм выполняется после каждого успешного хода, пока игра не завершится победой или ничьей.
